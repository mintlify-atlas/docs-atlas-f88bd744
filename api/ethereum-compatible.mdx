Core Lane implements standard Ethereum JSON-RPC methods, providing compatibility with existing Ethereum tools and libraries.

## Account Methods

### eth_getBalance

Returns the balance of the account at the given address.

```bash
curl -X POST http://127.0.0.1:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_getBalance",
    "params": ["0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266", "latest"],
    "id": 1
  }'
```

<ParamField path="params[0]" type="string" required>
  Address to check (20-byte hex string with 0x prefix)
</ParamField>

<ParamField path="params[1]" type="string" required>
  Block parameter: "latest", "earliest", "pending", or block number (hex)
</ParamField>

<ResponseField name="result" type="string">
  Account balance in wei (hex-encoded)
</ResponseField>

**Example Response:**
```json
{
  "jsonrpc": "2.0",
  "result": "0xde0b6b3a7640000",
  "id": 1
}
```

## Transaction Methods

### eth_sendRawTransaction

Sends a signed transaction to the network.

```bash
curl -X POST http://127.0.0.1:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_sendRawTransaction",
    "params": ["0x02f8...signed_tx_data..."],
    "id": 1
  }'
```

<ParamField path="params[0]" type="string" required>
  Signed transaction data (hex-encoded with 0x prefix)
</ParamField>

<ResponseField name="result" type="string">
  Transaction hash (32-byte hex)
</ResponseField>

**Example Response:**
```json
{
  "jsonrpc": "2.0",
  "result": "0x7d2a2c7e2e7c58e7f9e7d2a2c7e2e7c58e7f9e7d2a2c7e2e7c58e7f9e7d2a2c",
  "id": 1
}
```

<Info>
Depending on node configuration, transactions may be:
- Submitted to Bitcoin DA (if Bitcoin client is configured)
- Forwarded to a sequencer RPC (if `--sequencer-rpc-url` is set)
- Submitted to Espresso (if Espresso submit URL is configured)
- Forwarded to upstream Core Lane RPC (in derived mode)
</Info>

---

### eth_sendTransaction

Sends a transaction from an account (requires unlocked account).

<Warning>
This method is not yet implemented. Use `eth_sendRawTransaction` with pre-signed transactions instead.
</Warning>

### eth_getTransactionByHash

Returns transaction information by hash.

```bash
curl -X POST http://127.0.0.1:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_getTransactionByHash",
    "params": ["0x7d2a2c7e2e7c58e7f9e7d2a2c7e2e7c58e7f9e7d2a2c7e2e7c58e7f9e7d2a2c"],
    "id": 1
  }'
```

<ParamField path="params[0]" type="string" required>
  Transaction hash (32-byte hex)
</ParamField>

<ResponseField name="result" type="object | null">
  Transaction object or null if not found
  <Expandable title="Transaction Object Fields">
    <ResponseField name="hash" type="string">
      Transaction hash
    </ResponseField>
    <ResponseField name="nonce" type="string">
      Number of transactions from sender (hex)
    </ResponseField>
    <ResponseField name="blockHash" type="string">
      Hash of containing block
    </ResponseField>
    <ResponseField name="blockNumber" type="string">
      Block number (hex)
    </ResponseField>
    <ResponseField name="transactionIndex" type="string">
      Position in block (hex)
    </ResponseField>
    <ResponseField name="from" type="string">
      Sender address
    </ResponseField>
    <ResponseField name="to" type="string">
      Recipient address (or "null" for contract creation)
    </ResponseField>
    <ResponseField name="value" type="string">
      Transfer value in wei (hex)
    </ResponseField>
    <ResponseField name="gas" type="string">
      Gas limit (hex)
    </ResponseField>
    <ResponseField name="gasPrice" type="string">
      Gas price in wei (hex, for Legacy/EIP-2930)
    </ResponseField>
    <ResponseField name="maxFeePerGas" type="string">
      Max fee per gas (hex, for EIP-1559)
    </ResponseField>
    <ResponseField name="maxPriorityFeePerGas" type="string">
      Max priority fee per gas (hex, for EIP-1559)
    </ResponseField>
    <ResponseField name="input" type="string">
      Transaction data (hex)
    </ResponseField>
    <ResponseField name="v" type="string">
      ECDSA recovery id (hex)
    </ResponseField>
    <ResponseField name="r" type="string">
      ECDSA signature r (hex)
    </ResponseField>
    <ResponseField name="s" type="string">
      ECDSA signature s (hex)
    </ResponseField>
    <ResponseField name="type" type="string">
      Transaction type: "0x0" (Legacy), "0x1" (EIP-2930), "0x2" (EIP-1559)
    </ResponseField>
    <ResponseField name="chainId" type="string">
      Chain ID (hex)
    </ResponseField>
    <ResponseField name="accessList" type="array">
      Access list (for EIP-2930 and EIP-1559)
    </ResponseField>
  </Expandable>
</ResponseField>

```bash

  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_call",
    "params": [{
      "to": "0x0000000000000000000000000000000000000045",
      "data": "0x..."
    }, "latest"],
    "id": 1
  }'
```

<ParamField path="params[0]" type="object" required>
  Transaction call object
  <Expandable title="Call Object Fields">
    <ParamField path="from" type="string">
      Sender address (optional)
    </ParamField>
    <ParamField path="to" type="string">
      Target address
    </ParamField>
    <ParamField path="data" type="string">
      Encoded function call (hex)
    </ParamField>
    <ParamField path="value" type="string">
      Value to send (hex, optional)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="params[1]" type="string" required>
  Block parameter
</ParamField>

<ResponseField name="result" type="string">
  Call result (hex-encoded bytes)
</ResponseField>
